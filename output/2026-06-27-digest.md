# AI Trend Digest — 2026-06-27

## 이번 주 핵심 트렌드

에이전트 하네스 설계가 학계 주류로 자리잡았습니다 — OS 수준 정책 집행, 레거시 워크플로우 전환, post-training과의 상호작용까지 다양한 각도의 논문이 동시에 등장했습니다. 산업적으로는 미국 정부가 GPT-5.6과 Anthropic Mythos의 접근을 직접 통제하기 시작하면서 AI 모델이 국가 인프라 수준의 자산으로 격상됐습니다. 에이전트 보안과 프라이버시도 독립 연구 영역으로 빠르게 부상 중입니다.

## Harness & Orchestration

- **[The Interplay of Harness Design and Post-Training in LLM Agents](http://arxiv.org/abs/2606.25447v1)** `2026-06-24`
  하네스(도구 노출 방식, 관찰 정보 구성)가 post-training 결과에 직접 영향을 준다는 연구. 하네스를 학습 변수로 다뤄야 한다는 주장.

- **[ActPlane: Programmable OS-Level Policy Enforcement for Agent Harnesses](http://arxiv.org/abs/2606.25189v1)** `2026-06-23`
  자연어 정책("커밋 전 테스트 실행")을 OS 수준 시스템 액션으로 브리징하는 런타임.

- **[A Process Harness for Uplifting Legacy Workflows to Agentic BPM](http://arxiv.org/abs/2606.27188v1)** `2026-06-25`
  기존 워크플로우 엔진을 교체하지 않고 agentic 레이어를 감싸는 "process harness" 패턴.

- **[NOVA: Verification-Aware Agent Harness for Architecture Evolution](http://arxiv.org/abs/2606.27243v1)** `2026-06-25`
  산업용 추천 시스템 아키텍처 진화를 위한 검증 인식 하네스.

- **[AOHP: OS-Level Agent Harness for Personalized, Efficient and Secure Interaction](http://arxiv.org/abs/2606.23449v1)** `2026-06-22`
  AI 에이전트를 위한 OS 수준 오픈소스 하네스. 기존 앱 중심 OS의 한계를 극복하기 위한 새 시스템 패러다임 제안.

## Loop Engineering

- **[Semantic Early-Stopping for Iterative LLM Agent Loops](http://arxiv.org/abs/2606.27009v1)** `2026-06-25`
  고정 iteration cap 대신 임베딩 cosine distance로 루프 종료를 결정하는 semantic kill-switch. 쉬운 입력엔 토큰 절약, 어려운 입력엔 더 많은 반복 허용.

- **[Where Do CoT Training Gains Land in LLM based Agents?](http://arxiv.org/abs/2606.26935v1)** `2026-06-25`
  CoT 학습이 "추론을 통한 행동 변화"를 개선하는지 vs "직접 행동 예측"을 개선하는지 분리 분석.

- **[Don't Blindly Trust It: How Unreliable Feedback Breaks Tool-Using LLM Agents](http://arxiv.org/abs/2606.21409v1)** `2026-06-19`
  루프 내 신뢰할 수 없는 피드백이 오히려 피드백 없는 것보다 성능을 저하시킴을 실증.
