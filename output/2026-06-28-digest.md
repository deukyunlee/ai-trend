## Harness & Orchestration

- **[The Interplay of Harness Design and Post-Training in LLM Agents](http://arxiv.org/abs/2606.25447v1)**
  하네스(도구 노출 방식, 설명 형태, 관찰 정보 구성)가 post-training 결과에 직접 영향을 준다는 연구.

- **[ActPlane: Programmable OS-Level Policy Enforcement for Agent Harnesses](http://arxiv.org/abs/2606.25189v1)**
  자연어 정책을 OS 수준 시스템 액션으로 브리징하는 런타임. 기존 per-call 가드레일의 한계를 지적.

- **[A Process Harness for Uplifting Legacy Workflows to Agentic BPM](http://arxiv.org/abs/2606.27188v1)**
  기존 워크플로우 엔진을 교체하지 않고 agentic 레이어를 감싸는 "process harness" 패턴.

- **[NOVA: A Verification-Aware Agent Harness for Architecture Evolution](http://arxiv.org/abs/2606.27243v1)**
  산업용 추천 시스템의 아키텍처 진화를 위한 검증 인식 하네스.

- **[A Deterministic Control Plane for LLM Coding Agents](http://arxiv.org/abs/2606.26924v1)**
  10,008개 GitHub 레포 분석 — 에이전트 설정 파일의 10.1%가 SHA-256 중복.

## Loop Engineering

- **[Semantic Early-Stopping for Iterative LLM Agent Loops](http://arxiv.org/abs/2606.27009v1)**
  고정 iteration cap 대신 임베딩 cosine distance로 루프 종료를 결정하는 semantic kill-switch.

- **[Don't Blindly Trust It: How Unreliable Feedback Breaks Tool-Using LLM Agents](http://arxiv.org/abs/2606.21409v1)**
  루프 내 피드백이 신뢰할 수 없을 때 오히려 피드백 없는 것보다 성능이 나빠짐을 실증.

- **[Where Do CoT Training Gains Land in LLM based Agents?](http://arxiv.org/abs/2606.26935v1)**
  CoT 학습이 "추론을 통한 행동 변화"를 개선하는지 vs "직접 행동 예측"을 개선하는지 분리 분석.
